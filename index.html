Object oriented programming is the most common method of coding today. OOP models a system that is a collection of of objects, where each object something in that system. for example Java and C++ are both object oreinted programing languages. In OOP, Objects are key constructss in our programming, and these objects contain properties and functionality.

Terminology: - Classes vs. Objects

Classes

- is a blueprint, in which you create your properties and functionality. allow the creation of objects that store internal data, and expose behavior to users of those objects. they also allow private fields, which means data can be hidden from users if

Objects:

- is the actual product developed from the blueprint. an object is an instance if a clss. So, a class is a template for objects, and an object is an instance of a class. many use these terms interchangeably. They are sepereate in JavaScript.

JavaScript and OOP:

JavaScript has features of those described above; however, as we stated earlier, classes and objects in oop are two seperate constructs. in JS objects are often created without any seperate class definition, by either using a function or an object literal. Thus, programming objects is mch more light-wieght than in class-based OOP.

When to create a new class?

Generally classes are used for one purpose. for example, if you are building a card game program. you would have a class for every aspect. one for the dealer, one for the deck of cards, one for the players in the game, a class with properties and functionalities each designed to function specific to its task.

The Four Pillars of OOP

Abstraction, Encapsulation, Inheritance, & Polymorphism

Abstraction:

An abstraction is a way of hiding the implementation details and showing only the functionality to the users. In other words, it ignores the irrelevant details and shows only the required one.

Encapsulation:

this means wrapping up data and methods together into a single unit like a class. it is also built on the idea of data hiding. The variables of a class will be hidden from other classes and can be accessed only through the methods of thier current class.

Inheritance:

This is key when reusing code in an object-oriented program. It's possible to reuse properties and methods when a new class is created from an existing class. This is used if you have multiple classes with similar properties.

Polymorphism:

The polymorphism is a core concept of an object-oriented paradigm that provides a way to perform a single action in different forms. It provides an ability to call the same method on different JavaScript objects. Polymorphism means the same function with different signatures is called many times. In real life, for example, a boy at the same time may be a student, a class monitor, etc. So a boy can perform different operations at the same time. This is called polymorphism.
